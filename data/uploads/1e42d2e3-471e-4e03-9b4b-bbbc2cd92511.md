# 1. Project Overview and Goals

We aim to develop a **self-hosted, white-label WhatsApp CRM platform** modeled on DoubleTick, engineered with:

- Superior performance  
- Deeper automation  
- Enterprise scalability  

The platform will integrate directly with:

- Chatwoot Self Host  
- Meta WhatsApp Cloud API  

This ensures:

- Full data ownership  
- Multi-company management  
- Centralized CRM workflows  

The MVP will deliver:

- Core communication features  
- CRM tracking  
- Message automation  
- Templates  
- Broadcast  
- Analytics  
- User roles  

The architecture is designed to evolve into a full-scale SaaS.

## Core Objectives

1. Build a DoubleTick-like communication & CRM system enhanced with modern features.  
2. Provide multi-company tenant support via a single admin portal.  
3. Ensure all data remains on our servers (security + compliance).  
4. Deploy MVP within 8–10 weeks using scalable microservice-ready architecture.  
5. Establish long-term roadmap for AI, omni-channel, workflow automation, and marketplace integrations.

# Workflows

## Messaging Flow

1. Customer sends WhatsApp message  
2. Meta → Webhook (Message Receiver)  
3. Processor → Store in database  
4. Notify frontend (WebSocket)  
5. Agent replies  
6. Backend → Meta Cloud → Customer  

---

## Lead Creation Flow

WhatsApp message →  
No existing contact →  
Auto-create lead →  
Assign pipeline stage →  
Agent follow-up →  
Status updates  

---

## Template Approval Flow

Admin uploads template →  
Backend pushes to Meta →  
Meta approves →  
Template becomes available in broadcast panel  

---

## Broadcast Flow

Select audience →  
Choose template →  
Schedule →  
Worker processes queue →  
Send →  
Log analytics

# 9. Chatwoot Role in This Project

Chatwoot will be used **only as a core messaging engine** within a larger custom-built system.

## What This Means

We DO NOT:

- Use Chatwoot as a full product  
- Keep default Chatwoot UI  
- Rely on limited built-in workflows  

We ONLY use it for:

- WhatsApp Inbox  
- Real-time messaging  
- Agent routing  
- WebSocket events  
- Conversation storage  
- Channel integrations  

Everything else is custom-built:

- CRM  
- Pipelines  
- Companies  
- Workflows  
- Broadcast studio  
- Automation engine  
- White-label portal  
- Billing  
- Dashboard

# Why We Use Chatwoot Strategically

It solves complex messaging challenges that would otherwise require 8–12 weeks of development:

- Real-time WebSocket syncing  
- Multi-agent messaging support  
- WhatsApp Cloud API receiver integration  
- Conversation thread models  
- Channel adapters  
- Reliable backend event queue  
- Chat UI foundation  

This gives us a battle-tested messaging backbone.
